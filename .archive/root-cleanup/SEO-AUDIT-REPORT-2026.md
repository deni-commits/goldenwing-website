# GoldenWing Website - SEO Audit Report
**Date:** January 29, 2026
**Domain:** https://goldenwing.at
**Locales:** German (de), English (en), Russian (ru - partial)
**Framework:** Next.js 15+ with Payload CMS

---

## Executive Summary

The GoldenWing website has a **solid SEO foundation** with good implementation of technical SEO, schema markup, and internationalization. However, there are several issues that need attention to optimize for search engines and improve overall SEO performance.

**Overall Status:** ⚠️ **GOOD with CRITICAL & MINOR ISSUES to fix**

---

## 1. META TAGS ANALYSIS

### ✅ STRENGTHS
- **Comprehensive Meta Tag Implementation:** All pages have `title`, `description`, `openGraph`, and `twitter` meta tags
- **Dynamic Generation:** Meta tags are dynamically generated from CMS data with proper fallbacks
- **Multi-locale Support:** Each locale has appropriately translated meta tags (de/en/ru)
- **Open Graph Tags:** Present with proper image URLs (1200x630px - correct dimensions)
- **Twitter Cards:** Implemented with `summary_large_image` format

### ❌ CRITICAL ISSUES

#### Issue #1.1: Missing/Empty Meta Descriptions
**File:** Multiple pages
**Severity:** HIGH
**Details:**
- Some pages may have empty `metaDescription` fields from CMS
- Example: Service pages, landing pages without CMS entries
- Character count not validated (should be 120-160 characters)

**Recommendation:**
- Add validation in metadata generation to ensure descriptions are 120-160 characters
- Implement default descriptions for all pages without CMS content
- Example fix in `src/app/[locale]/(marketing)/page.tsx`:
```typescript
const description = seo?.metaDescription?.slice(0, 160) ||
  'Default fallback description (160 chars max)';
```

#### Issue #1.2: Inconsistent OpenGraph Image Usage
**File:** Multiple page.tsx files
**Severity:** MEDIUM
**Details:**
- Most pages use generic `/og-image.jpg` (354KB - very large)
- No page-specific OG images for landing pages/services
- OG image size not optimized for web (354KB is excessive)

**Recommendation:**
- Resize and optimize main OG image (target: 100-150KB)
- Generate dynamic OG images for landing pages using the existing `opengraph-image.tsx` components
- Implement per-page OG image support in CMS

#### Issue #1.3: Meta Viewport Tag Location
**File:** `src/app/layout.tsx`
**Severity:** LOW
**Details:**
- Root layout is minimal and doesn't define viewport meta tag
- This should be in the marketing layout or global metadata

**Recommendation:**
- Verify viewport tag is being generated by Next.js automatically (it should be)
- Explicitly add if needed in marketing layout's metadata export

## 2. SCHEMA MARKUP (JSON-LD) ANALYSIS

### ✅ STRENGTHS
- **Comprehensive Schema Implementation:** Multiple schema types implemented
- **Proper JSON-LD Format:** All schemas use correct `@context` and `application/ld+json`
- **Organization Schema:** LocalBusiness schema with proper contact details and geo coordinates
- **Multiple Schema Types:** Service, Person, Team, Review, AggregateRating, SoftwareApplication, Credentials schemas available
- **E-E-A-T Optimization:** Credentials and Person schemas support expertise signals

**File:** `/src/components/seo/schemas.tsx`

### ⚠️ MEDIUM ISSUES

#### Issue #2.1: Missing/Incomplete Schema on Key Pages
**Severity:** MEDIUM
**Details:**
- Homepage (`page.tsx`) uses `ReviewSchema` and `CredentialsSchema` but not consistently
- Service pages may not all have `ServiceSchema` markup
- FAQ pages missing `FAQPage` schema type
- Blog pages missing `NewsArticle` or `BlogPosting` schema

**Recommendation:**
- Add `FAQPage` schema to `/haeufige-fragen` page:
```typescript
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": questions.map(q => ({
    "@type": "Question",
    "name": q.title,
    "acceptedAnswer": {
      "@type": "Answer",
      "text": q.answer
    }
  }))
}
```
- Add `BlogPosting` or `NewsArticle` schema to blog pages:
```typescript
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": post.title,
  "description": post.excerpt,
  "image": post.image,
  "datePublished": post.publishedAt,
  "author": { "@type": "Person", "name": post.author },
  "isPartOf": { "@type": "Blog", "name": "GoldenWing Blog" }
}
```

#### Issue #2.2: Missing BreadcrumbList Schema on Dynamic Pages
**Severity:** MEDIUM
**Details:**
- `BreadcrumbSchema` component exists but not used on all dynamic routes
- Service pages with sub-services should have breadcrumbs
- Missing on: `/leistungen/[slug]/[subslug]`, reference category pages

**Recommendation:**
- Add breadcrumb schema to all multi-level pages:
```typescript
<BreadcrumbSchema items={[
  { name: 'Home', url: '/' },
  { name: 'Services', url: '/leistungen' },
  { name: serviceName, url: `/leistungen/${slug}` },
  { name: subServiceName, url: `/leistungen/${slug}/${subslug}` }
]} />
```

#### Issue #2.3: LocalBusiness Schema Missing Key Fields
**Severity:** LOW
**Details:**
- Current schema has address and geo for Vienna only
- Missing: `sameAs` (social media profiles), `priceRange` in some contexts
- No support for multiple office locations (Dubai, Roseville)

**Recommendation:**
- Extend LocalBusiness schema to include all office locations:
```typescript
const officeLocations = [
  {
    name: 'Vienna Headquarters',
    address: { /* Vienna address */ },
    geo: { /* Vienna coords */ }
  },
  {
    name: 'Dubai Office',
    address: { /* Dubai address */ },
    geo: { /* Dubai coords */ }
  }

// Add sameAs for social profiles
sameAs: [
  'https://www.linkedin.com/company/goldenwing',
  'https://www.instagram.com/goldenwing',
  // etc.
]
```

#### Issue #2.4: Missing CredentialsSchema Validation
**Severity:** LOW
**Details:**
- `CredentialsSchema` is defined but may not be used on appropriate pages
- No validation that credentials have valid `issuer` and `dateAwarded`

**Recommendation:**
- Use credentials schema on team member pages and about page
- Validate credential data structure in CMS

## 3. SITEMAP CONFIGURATION ANALYSIS

### ✅ STRENGTHS
- **Comprehensive Sitemap:** `src/app/sitemap.ts` generates dynamic sitemap.xml
- **Multi-locale Support:** Includes both DE and EN versions with correct hreflang alternates
- **Priority Strategy:** E-E-A-T optimized with thoughtful priority levels
- **Dynamic Content:** Includes blog posts, projects, services from CMS
- **Change Frequency:** Properly set based on content type
- **Last Modified:** Dynamically updated from CMS data

### ⚠️ ISSUES

#### Issue #3.1: Sitemap Size & Index Implementation
**File:** `/src/app/sitemap.ts`
**Severity:** MEDIUM
**Details:**
- Single sitemap.xml file generated (may exceed 50MB limit if many pages)
- `next-sitemap.config.js` has `generateIndexSitemap: false`
- No sitemap index (sitemap_index.xml) for large-scale sitemaps

**Recommendation:**
- Implement sitemap indexing if total URLs exceed 50,000:
```typescript
// In sitemap.ts
const urlsPerSitemap = 45000; // Stay under 50MB limit
if (totalUrls > urlsPerSitemap) {
  // Return array of sitemap URLs instead of direct routes
  return [
    'https://goldenwing.at/sitemap-0.xml',
    'https://goldenwing.at/sitemap-1.xml'
  ];
}
```
- Or enable in `next-sitemap.config.js`:
```javascript
generateIndexSitemap: true,
sitemapSize: 45000,
```

#### Issue #3.2: Missing Russian (ru) Locale in Sitemap
**Severity:** MEDIUM
**Details:**
- Sitemap only includes DE and EN versions
- Russian locale (ru) is available but not in sitemap
- Configuration in `src/i18n/config.ts`: `visibleLocales = ['de', 'en']` (ru hidden)

**Recommendation:**
- Add Russian URLs to sitemap.ts:
```typescript
// Add RU version alongside DE/EN
routes.push({
  url: `${BASE_URL}/ru${translatePath(route.path)}`,
  lastModified: new Date(),
  changeFrequency: route.changeFrequency,
  priority: route.priority * 0.9,
  alternates: {
    languages: {
      de: `${BASE_URL}${dePath}`,
      en: `${BASE_URL}${enPath}`,
      ru: `${BASE_URL}/ru${ruPath}`
    }
  }
})
```
- Or explicitly exclude RU if not ready with a comment

#### Issue #3.3: Sitemap vs robots.txt Mismatch
**Severity:** LOW
**Details:**
- `robots.txt` references `https://goldenwing.at/sitemap.xml`
- But generated by Next.js (should be auto-handled)
- Two sitemap configs: `src/app/sitemap.ts` AND `next-sitemap.config.js`

**Recommendation:**
- Use ONLY `src/app/sitemap.ts` (Next.js native - recommended)
- Remove or disable `next-sitemap.config.js` to avoid conflicts
- Keep robots.txt pointing to auto-generated sitemap

#### Issue #3.4: Incorrect Sitemap Priorities
**Severity:** LOW
**Details:**
- Some priorities may be too high (multiple pages at 0.9)
- Landing pages ranked equal to homepage sections
- AEO listicle pages at 0.9 same as main services

**Recommendation:**
- Refine priority hierarchy:
```
1.0  → Homepage only
0.95 → Main conversion pages (Kontakt, Leistungen)
0.9  → E-E-A-T pages (Team, About)
0.85 → AEO listicle pages + major landing pages
0.8  → Service pages
0.75 → Sub-services + Projects
0.7  → Blog posts + Landing pages (secondary)
0.6  → Blog categories + Supporting pages
0.5  → Resources/Downloads
0.3  → Legal pages
0.2  → Legal subsections
```

**Details:**
- Some pages may have multiple H1 tags (not technically wrong in HTML5 but SEO anti-pattern)

#### Issue #8.2: H1 Content Not Always Matching Meta Title
**Severity:** MEDIUM
**Details:**
- H1 and meta title should be related but often differ significantly
- May confuse search engines about page topic

**Recommendation:**
- Ensure H1 and meta title are closely aligned:
```typescript
// In metadata
title: 'Web Design Vienna | Creative Agency',
```

**Severity:** LOW

```
H1 (Main topic)
  H2 (Major section)
    H3 (Subsection)
    H3 (Another subsection)

```

#### Issue #8.4: Semantic Headings Using CSS Classes
